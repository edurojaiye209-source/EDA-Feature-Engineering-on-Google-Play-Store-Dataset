1. Project Title
EDA & Feature Engineering on Google Play Store Dataset

2. Project Description
This project focuses on performing Exploratory Data Analysis (EDA) and Feature Engineering on the Google Play Store dataset to uncover insights about app ratings, installs, pricing, and categories.

3. Objectives 
Understand factors affecting app ratings
Identify trends in app installs and pricing
Handle missing values and outliers
Prepare dataset for machine learning

4. Dataset Information
Source (Kaggle)
Number of rows & columns(10346, 15)
Key columns (Rating, Reviews, Installs, Category, Price)

5. Data Cleaning Steps
Removed duplicates
Handled missing values
Converted data types (e.g. installs to numeric)
Fixed inconsistent values

6. EDA (Exploratory Data Analysis)
Distribution of ratings
Most popular app categories
Relationship between installs and ratings
Price vs installs

7. Feature Engineering
Converted categorical data (e.g Category → numbers)
Created new features
Handled skewness/outliers

8. Tools Used
Python
Pandas
Matplotlib / Seaborn

9. Key Insights
Free apps have higher installs than paid apps
Most apps fall between rating 4.0–4.5
Certain categories dominate the market

10. Conclusion
The dataset was cleaned, analyzed, and transformed into a structured format suitable for machine learning models.
